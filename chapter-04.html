Chapter 4 · Foundation

Computer Software

The programs that bring the hardware to life: the apps people use to do their work, and the system software that runs the machine underneath. The "mind" to Chapter 3's "body."

Good to know: like hardware, software is a background chapter rarely asked about directly in the exam. Read it for comfort with the vocabulary. The one idea that does keep coming up elsewhere is the build-vs-buy-vs-rent decision at the end.
In plain English - the whole chapter as one story

Our running example: BrewHaus. Chapter 3 was the physical machine. This chapter is everything that runs on that machine: the software.

1. All software splits into two big families

Application software does a job for you (the app that takes a coffee order). System software runs the computer itself (the operating system that makes the app work at all). Everything else in the chapter sits under one of those two.

2. Application software: the tools you actually touch

Some apps are general-purpose, useful to anyone (word processors, spreadsheets, email, browsers). Others are built for a specific job (a coffee-shop point-of-sale app, an accounting package).

3. System software: the stuff working quietly underneath

The most important piece is the operating system, the master program that has to be running before anything else can. It manages the hardware, the files, and the tasks, so your apps do not have to.

4. Software has to be written, and there are easier and harder ways

Programming languages range from raw machine code (all 0s and 1s) up to languages that read almost like English. The closer to English, the easier for humans, the more translation the computer does.

5. You do not always have to write or even buy it

BrewHaus could build its own loyalty app, buy a ready-made one, or simply rent one over the internet and pay monthly. That last option, renting software as a service, is the big modern shift.

How the chapter connects: software divides into application and system → application software is your everyday tools → system software (led by the operating system) runs the machine → all of it is written in programming languages of varying friendliness → some of it is free and open-source → and increasingly you rent it over the internet rather than owning it. That is the software landscape.

Section 1

The two kinds of software

When people say "software," what are they really talking about?

Everything called software falls into one of two families, and keeping them straight makes the whole chapter simple:

🧰

Application software

Performs information-processing tasks for the end user. The programs that do your work: writing, calculating, ordering, emailing.

⚙️

System software

Manages and supports the operations of the computer itself. The programs that run the machine so the apps can run on it.

A simple test: if it helps a person get a job done, it is application software. If it helps the computer function, it is system software.

Section 2

Application software: the tools you use

What kinds of apps are there?

Application software splits again into two groups:

  • General-purpose programs - useful to almost anyone: word processors, spreadsheets, presentation graphics, database managers, email, web browsers, personal information managers (calendar and contacts), and groupware (tools for teams to collaborate).
  • Application-specific programs - built for a particular job. In business these include the big four you will meet again in Chapter 7: CRM (customer relationships), ERP (running the whole enterprise), SCM (supply chain), and web-based e-commerce.

Two general-purpose tools worth a special mention, because they show up everywhere:

  • Spreadsheets - rows and columns of data with formulas linking them. Their superpower is answering "what if?" questions: change one number and watch everything recalculate. BrewHaus uses one to model "what if I open a fourth shop?"
  • Web browsers - increasingly the universal launchpad: you do email, searches and apps all inside the browser. Search engines reached through the browser have become indispensable.
Section 3

Suites versus integrated packages

Why does office software come in a big bundle?

Two ways to package productivity tools, with a clear trade-off:

 Software suiteIntegrated package
What it isSeveral full, separate programs bundled together (e.g. a word processor + spreadsheet + presentation + database + calendar).One program that does several jobs in a simpler way.
UpsideCheaper than buying each alone; shared look-and-feel; programs work well together.Many functions for a low price; light on disk space; often pre-installed.
DownsideYou never use all the features, and it eats disk space ("bloatware").Limited power.

The classic suite is the office bundle (a word processor, a spreadsheet, a presentation tool, a database and a calendar app, all sharing one style). An integrated package trades power for simplicity and price.

Section 4

System software: working underneath

If application software does my work, what is system software doing?

System software sits as a middle layer between your apps and the bare hardware. You never touch the hardware directly; the layers handle that for you:

System software itself comes in two groups:

  • System management programs - run the resources: the operating system, network management, database management systems, utilities (housekeeping like backup and virus protection), performance and security monitors, and application servers that connect apps to the web.
  • System development programs - help people build software: language translators, programming editors and tools, and CASE tools (which combine many programming tools in one).
Section 5

The operating system: the master program

What is the single most important piece of software on any computer?

The operating system (OS). It must be loaded and running before any other program can do anything. It is the manager that handles four core jobs, plus the way you interact with the machine:

OS jobWhat it manages
Resource managementThe use of the hardware (processor, memory, devices).
Task managementGetting the computer's tasks done, juggling several at once.
File managementOrganising data and program files.
Utilities and supportA range of background support services.

The user interface is how you give it instructions: old command-driven (type commands), menu-driven (pick from lists), or the modern graphical user interface (GUI) (click icons and windows). The familiar operating systems are Windows, Unix (portable, runs on everything), Linux (low-cost, open-source, Unix-like) and Mac OS X (Apple's).

Section 6

Programming languages

How is software actually written, and why are there so many languages?

Software is written in programming languages, and they come in "generations" that climb from the computer's native tongue toward human language:

GenerationWhat it looks likeFriendliness
1st: Machine languageRaw binary (0s and 1s) unique to each computer.Hardest for humans, closest to the hardware.
2nd: Assembler languageShort symbolic abbreviations (mnemonics) instead of binary.A little easier; needs an "assembler" to translate.
3rd: High-level languageReadable statements and arithmetic (BASIC, COBOL, FORTRAN).Much friendlier; needs a compiler or interpreter.
4th: 4GLNear-English: you say what you want, not how ("sum the following numbers").Friendliest; the computer works out the steps.

Two more ideas you should recognise:

  • Object-oriented languages (Java, C++, Visual Basic) are today's most popular. They bundle data together with the operations that act on it into reusable "objects," so a piece built for one program can be reused in another.
  • Web languages: HTML describes how a web page looks; XML labels what the data means so different systems can share it; and Java runs the same little program on any computer.
Translators turn human-written code into machine code. An assembler translates assembly language; a compiler translates a whole high-level program before it runs; an interpreter translates and runs it one line at a time.
Section 7

Open source software

What does "open source" actually mean, and is it just "free software"?

Open source software ships with its source code, so anyone can read, modify and redistribute it. Because many people can improve it, it can evolve very fast. But "open source" is not the same as "free":

  • It is not simply shareware, freeware, or public-domain software.
  • It is copyrighted and comes with license terms, and it can even carry a fee for packaging, distribution or support.

Familiar examples include the Linux operating system and OpenOffice. The big idea for a business: open source can cut licensing costs and avoid lock-in to one vendor, at the price of taking on more responsibility for support yourself.

Section 8

Build, buy, or rent it (SaaS)

When BrewHaus needs new software, what are its options? (This part does matter later.)

There are three broad ways to get software, and choosing between them is a recurring business decision:

🔨

Build it

Write your own, in-house or outsourced. Fits you exactly, but costly and slow, and you maintain it forever.

🛒

Buy it

Purchase a ready-made package. Faster and cheaper, but you adapt to how it works.

☁️

Rent it (SaaS)

Use software over the internet for a monthly fee. No hardware to run, scales instantly, cancel anytime.

That third option is the modern shift. Software-as-a-service (SaaS), delivered through cloud computing, means the software and even the hardware live on someone else's servers and you simply pay for what you use. A few key points:

  • It is pay-as-you-go, avoiding big upfront purchases.
  • Capacity scales up almost instantly when you grow.
  • Favour configuration over customisation: keeping the software standard keeps you agile; bending it heavily to your needs adds cost and risk.
  • Always keep a way to back up your data in case the provider disappears.
One licensing point people miss: when you "buy" packaged software, you do not own it. You buy a licence to use it; the vendor keeps the ownership rights.
